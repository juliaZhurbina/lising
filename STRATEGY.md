# Стратегия разработки Telegram-бота с GigaChat API

## Анализ требований

### База данных (Книга5.xlsx)
- **Объем**: 2180 записей о заявках
- **Структура**: 27 колонок, включая:
  - ID заявки (например, Z25-1869607)
  - Статусы заявок
  - Даты (дата создания, дата изменения статуса)
  - Информация о клиентах и автомобилях
  - История изменений

### Задачи (Пункт 5 и расширенный функционал)

#### Основная задача (Пункт 5)
**Описание**: Автоматически изменять статус заявки по запросу пользователя.

**Пример запроса**: "измени статус заявки Z12345 на 15 ноября"

**Ожидаемое поведение**: 
- Бот должен изменить статус заявки
- Подтвердить изменение: "дата изменения статуса заявки Z12345 изменена на 10.11 на 15.11. Подтвердить изменение?"

#### Расширенный функционал

1. **Поиск информации по заявке**
   - Поиск заявки по ID, клиенту, автомобилю
   - Получение полной информации о заявке
   - История изменений и статусов
   - Примеры запросов:
     - "найди заявку Z25-1869607"
     - "покажи информацию о заявке Z12345"
     - "какой статус у заявки Z25-1869607?"

2. **Генерация контекста для звонков и встреч**
   - Анализ данных заявки через GigaChat
   - Формирование структурированного контекста:
     - Информация о клиенте
     - История взаимодействий
     - Текущий статус и важные детали
     - Рекомендации для разговора
   - Примеры запросов:
     - "подготовь контекст для звонка по заявке Z12345"
     - "что нужно знать перед встречей с клиентом заявки Z25-1869607?"
     - "сгенерируй краткую справку по заявке Z12345"

3. **Работа с задачами**
   - Создание задач по заявкам
   - Напоминания о важных датах
   - Отслеживание выполнения задач
   - Примеры запросов:
     - "создай задачу позвонить клиенту заявки Z12345 завтра"
     - "напомни мне о встрече по заявке Z25-1869607"
     - "какие задачи у меня по заявке Z12345?"

**Технологии**: RPA + NLP подход с использованием GigaChat API

---

## Архитектура решения

### Компоненты системы

1. **Telegram Bot** (python-telegram-bot или aiogram)
   - Прием сообщений от пользователей
   - Отправка ответов и запросов подтверждения
   - Обработка callback-кнопок для подтверждения

2. **GigaChat API Integration**
   - Понимание естественного языка (NLP)
   - Извлечение сущностей: ID заявки, дата, действие, тип запроса
   - Определение намерения пользователя (intent classification)
   - Генерация подтверждающих сообщений
   - Генерация контекста для звонков/встреч
   - Анализ и структурирование данных заявки

3. **CRM Data Layer**
   - Чтение данных из Excel (или переход на БД)
   - Поиск заявки по ID, клиенту, автомобилю
   - Получение полной информации о заявке
   - Обновление статуса заявки
   - Сохранение истории изменений
   - Извлечение релевантных данных для контекста

4. **Business Logic Layer**
   - Валидация запросов
   - Проверка прав доступа
   - Логика изменения статусов
   - Обработка подтверждений
   - Поиск и фильтрация заявок
   - Генерация контекста для коммуникаций
   - Управление задачами и напоминаниями

5. **Task Management Layer** (новый компонент)
   - Создание задач по заявкам
   - Хранение задач (в памяти или БД)
   - Напоминания и уведомления
   - Отслеживание выполнения задач

## Этапы разработки

### Этап 1: Подготовка инфраструктуры
**Цель**: Настроить базовую структуру проекта

**Задачи**:
1. Создать структуру проекта
2. Установить зависимости:

   - `gigachat` (SDK для GigaChat API)

   - `openpyxl` (для чтения/записи Excel)
   - `python-dotenv` (для конфигурации)
3. Настроить конфигурацию (токены, пути к файлам)
4. Создать файл `.env` для секретов

**Результат**: Готовая структура проекта с зависимостями

---

### Этап 2: Интеграция с базой данных
**Цель**: Реализовать работу с Excel-файлом

**Задачи**:
1. Создать класс `DatabaseManager`:
   - Метод `find_application(app_id)` - поиск заявки по ID
   - Метод `search_applications(query)` - поиск заявок по различным критериям (ID, клиент, автомобиль)
   - Метод `get_application_full_info(app_id)` - получение полной информации о заявке
   - Метод `get_application_status(app_id)` - получение текущего статуса
   - Метод `update_application_status(app_id, new_date, new_status)` - обновление статуса
   - Метод `get_application_history(app_id)` - получение истории изменений
   - Метод `get_application_context_data(app_id)` - получение данных для генерации контекста
2. Реализовать чтение Excel файла с кэшированием
3. Реализовать запись изменений обратно в Excel (или логирование)
4. Реализовать индексацию данных для быстрого поиска

**Результат**: Работающий модуль для работы с данными заявок с расширенными возможностями поиска

---

### Этап 3: Интеграция с GigaChat API
**Цель**: Настроить работу с GigaChat для NLP

**Задачи**:
1. Получить токен доступа к GigaChat API
2. Создать класс `GigaChatService`:
   - Метод `extract_intent(message)` - определение намерения пользователя (изменить статус, найти заявку, сгенерировать контекст, создать задачу)
   - Метод `extract_entities(message)` - извлечение ID заявки, даты, действия, параметров задачи
   - Метод `generate_confirmation_message(app_id, old_date, new_date)` - генерация подтверждения
   - Метод `generate_call_context(application_data)` - генерация контекста для звонка/встречи
   - Метод `format_application_info(application_data)` - форматирование информации о заявке
   - Метод `analyze_application_for_context(application_data)` - анализ заявки для подготовки контекста
3. Настроить промпты для:
   - Извлечения информации из запроса (intent + entities)
   - Генерации понятных ответов
   - Генерации структурированного контекста для коммуникаций
   - Анализа истории заявки для выявления важных моментов
4. Обработка ошибок и валидация ответов от API
5. Реализовать кэширование ответов GigaChat для одинаковых запросов

**Результат**: Работающая интеграция с GigaChat для понимания запросов и генерации контекста

---

### Этап 4: Разработка Telegram бота
**Цель**: Создать базового бота с обработкой команд

**Задачи**:
1. Создать класс `TelegramBot`:
   - Инициализация бота с токеном
   - Обработчик текстовых сообщений
   - Обработчик callback-кнопок (подтверждение/отмена, действия с задачами)
   - Обработчик inline-кнопок для быстрых действий
2. Реализовать команды:
   - `/start` - приветствие и инструкция
   - `/help` - справка по использованию с примерами
   - `/tasks` - список активных задач пользователя
   - `/search <query>` - быстрый поиск заявок
3. Реализовать обработку естественного языка:
   - Передача сообщения в GigaChat для анализа
   - Определение типа запроса (intent)
   - Извлечение сущностей (ID заявки, дата, параметры задачи)
   - Маршрутизация запроса к соответствующему обработчику
4. Реализовать интерактивные элементы:
   - Кнопки "Подтвердить" / "Отменить" для изменения статуса
   - Inline-кнопки для быстрых действий ("Показать контекст", "Создать задачу")
   - Клавиатура с основными командами
5. Реализовать форматирование ответов:
   - Красивое отображение информации о заявке
   - Структурированный вывод контекста для звонков
   - Список задач с возможностью отметки выполнения

**Результат**: Работающий Telegram бот с расширенной функциональностью и удобным интерфейсом

---

### Этап 5: Реализация бизнес-логики основных функций
**Цель**: Реализовать все основные функции бота

**Задачи**:

#### 5.1. Изменение статуса заявки
1. Создать класс `StatusChangeService`:
   - Метод `process_status_change_request(message)` - обработка запроса
   - Метод `validate_request(app_id, new_date)` - валидация данных
   - Метод `prepare_confirmation(app_id, old_date, new_date)` - подготовка подтверждения
   - Метод `execute_status_change(app_id, new_date)` - выполнение изменения
2. Реализовать workflow изменения статуса

#### 5.2. Поиск информации по заявке
1. Создать класс `ApplicationSearchService`:
   - Метод `process_search_request(message)` - обработка запроса поиска
   - Метод `search_by_id(app_id)` - поиск по ID
   - Метод `search_by_criteria(criteria)` - поиск по различным критериям
   - Метод `format_application_response(application_data)` - форматирование ответа
2. Реализовать различные форматы поиска:
   - По ID заявки
   - По клиенту
   - По автомобилю
   - По статусу

#### 5.3. Генерация контекста для звонков/встреч
1. Создать класс `ContextGenerationService`:
   - Метод `process_context_request(message)` - обработка запроса на контекст
   - Метод `gather_context_data(app_id)` - сбор данных для контекста
   - Метод `generate_call_context(application_data)` - генерация контекста через GigaChat
   - Метод `format_context_message(context_data)` - форматирование контекста
2. Реализовать структуру контекста:
   - Основная информация о заявке
   - История взаимодействий
   - Текущий статус и важные детали
   - Рекомендации для разговора
   - Ключевые моменты для обсуждения

#### 5.4. Работа с задачами
1. Создать класс `TaskManagementService`:
   - Метод `process_task_request(message)` - обработка запроса на задачу
   - Метод `create_task(app_id, task_description, due_date, user_id)` - создание задачи
   - Метод `get_user_tasks(user_id)` - получение задач пользователя
   - Метод `get_tasks_by_application(app_id)` - задачи по заявке
   - Метод `complete_task(task_id)` - отметка выполнения
   - Метод `send_reminders()` - отправка напоминаний
2. Реализовать хранение задач (SQLite или JSON файл)
3. Реализовать систему напоминаний

3. Обработка ошибок для всех функций:
   - Заявка не найдена
   - Некорректные данные
   - Ошибки при выполнении операций

**Результат**: Полностью работающий функционал со всеми основными возможностями

## Риски и решения

1. **Риск**: GigaChat API может неправильно извлекать сущности
   - **Решение**: Использовать структурированные промпты и валидацию ответов

2. **Риск**: Работа с Excel файлом может быть медленной
   - **Решение**: Кэширование данных в памяти, переход на БД при необходимости

3. **Риск**: Одновременное изменение одного файла несколькими пользователями
   - **Решение**: Блокировки файла или переход на БД с транзакциями

4. **Риск**: Ошибки в понимании естественного языка
   - **Решение**: Многоуровневая валидация и запрос подтверждения у пользователя
