# Примеры использования бота

## Сценарии работы с ботом

### 1. Изменение статуса заявки

**Запрос пользователя:**
```
"измени статус заявки Z25-1869607 на 15 ноября"
```

**Workflow:**
1. Бот анализирует запрос через GigaChat
2. Извлекает: `app_id=Z25-1869607`, `new_date=15.11.2025`
3. Находит заявку в базе данных
4. Получает текущий статус и дату
5. Генерирует подтверждение через GigaChat:
   ```
   "Дата изменения статуса заявки Z25-1869607 изменена с 10.11.2025 на 15.11.2025. 
   Подтвердить изменение?"
   ```
6. Отправляет сообщение с кнопками [Подтвердить] [Отменить]
7. При подтверждении → обновляет статус в БД
8. Отправляет подтверждение: "Статус успешно изменен!"

---

### 2. Поиск информации по заявке

**Запрос пользователя:**
```
"найди заявку Z25-1869607"
```

**Workflow:**
1. Бот определяет intent: `search_application`
2. Извлекает ID заявки: `Z25-1869607`
3. Ищет заявку в базе данных
4. Форматирует информацию через GigaChat:
   ```
   📋 Заявка Z25-1869607
   
   👤 Клиент: [Информация о клиенте]
   🚗 Автомобиль: GEELY MONJARO
   💰 Сумма: 3,594,110 руб.
   📅 Дата создания: 09.10.2025
   📊 Статус: [Текущий статус]
   📝 История изменений:
   - 10.10.2025: [Описание изменения]
   ```
5. Отправляет структурированное сообщение
6. Предлагает дополнительные действия: [Контекст для звонка] [Создать задачу]

---

### 3. Генерация контекста для звонка

**Запрос пользователя:**
```
"подготовь контекст для звонка по заявке Z25-1869607"
```

**Workflow:**
1. Бот определяет intent: `generate_call_context`
2. Извлекает ID заявки
3. Собирает все данные о заявке:
   - Основная информация
   - История взаимодействий
   - Текущий статус
   - Важные заметки
4. Передает данные в GigaChat для анализа
5. GigaChat генерирует структурированный контекст:
   ```
   📞 Контекст для звонка по заявке Z25-1869607
   
   🎯 Цель звонка:
   Обсудить статус заявки и уточнить детали по автомобилю GEELY MONJARO
   
   📋 Ключевая информация:
   - Клиент: [Имя клиента]
   - Автомобиль: GEELY MONJARO
   - Сумма сделки: 3,594,110 руб.
   - Дата создания: 09.10.2025
   
   📝 История взаимодействий:
   - 10.10.2025: Клиент уточнял сроки поставки
   - 14.10.2025: Назначена встреча на 15.10
   
   ⚠️ Важные моменты:
   - Клиент интересовался дополнительными опциями
   - Требуется подтверждение финансирования
   
   💡 Рекомендации:
   - Уточнить готовность к оформлению документов
   - Обсудить варианты оплаты
   - Предложить дополнительные услуги
   ```
6. Отправляет контекст пользователю
7. Предлагает сохранить как задачу или отправить на email

---

### 4. Создание задачи

**Запрос пользователя:**
```
"создай задачу позвонить клиенту заявки Z25-1869607 завтра в 14:00"
```

**Workflow:**
1. Бот определяет intent: `create_task`
2. Извлекает:
   - `app_id=Z25-1869607`
   - `task_description=позвонить клиенту`
   - `due_date=завтра в 14:00`
3. Находит заявку в БД
4. Создает задачу:
   ```json
   {
     "task_id": "task_12345",
     "user_id": 123456789,
     "app_id": "Z25-1869607",
     "description": "Позвонить клиенту",
     "due_date": "2025-01-27 14:00",
     "status": "pending",
     "created_at": "2025-01-26 10:30"
   }
   ```
5. Сохраняет задачу в хранилище
6. Отправляет подтверждение:
   ```
   ✅ Задача создана!
   
   📋 Позвонить клиенту заявки Z25-1869607
   ⏰ Напомнить: 27 января в 14:00
   
   [Отметить выполненной] [Изменить] [Удалить]
   ```
7. В указанное время отправляет напоминание

## Типы намерений (Intents)

Бот распознает следующие типы запросов:

1. **change_status** - Изменение статуса заявки
   - "измени статус", "обнови дату", "поменяй статус"

2. **search_application** - Поиск заявки
   - "найди заявку", "покажи информацию", "какой статус"

3. **generate_context** - Генерация контекста
   - "подготовь контекст", "что нужно знать", "справка для звонка"

4. **create_task** - Создание задачи
   - "создай задачу", "напомни", "добавь задачу"

5. **list_tasks** - Список задач
   - "мои задачи", "что нужно сделать", "покажи задачи"

6. **get_help** - Справка
   - "помощь", "что ты умеешь", "команды"

## Форматы ответов

### Структурированная информация о заявке
- Использует эмодзи для визуального разделения
- Группирует информацию по категориям
- Предлагает быстрые действия через кнопки

### Контекст для звонка
- Структурирован по разделам
- Включает рекомендации
- Выделяет важные моменты

### Подтверждения действий
- Четкие сообщения об успехе/ошибке
- Возможность отмены
- История изменений
